Four sidebars, one goal: quieter and cleaner

The current sidebar's search field sits flush against the note list with no breathing room, and dense rows make a long list feel noisy. These four alternatives all fix the search-to-list gap and pursue a more minimal reading — using Noschen's existing palette, no new colors — and differ in how far they go past that baseline. Switch between them below.

Quiet Rail

The refined-minimal baseline — same shape, real room to breathe.

Inset, rounded search field; a fixed 28px gap before the list starts; no dividers between rows; selection reads as a soft accent wash plus a leading bar, never color alone. Lowest risk, lowest effort, and it directly answers the "cramped" complaint.

Best forShipping the fix now, with no data-model changes. Trade-offGets visually noisier as note count grows past ~30.

Grouped Timeline

Mail.app-style date sections — chunking instead of raw padding.

Same inset search field, then a 24px gap into sticky "Today / Yesterday / This Week / Earlier" labels. Whitespace comes from grouping itself: every section boundary is a moment of rest, and rows inside a group can sit tighter since the label already carries "when." Scans better than Quiet Rail as the list grows.

Best forUsers who accumulate dozens of notes and need to scan, not just skim. Trade-offSearch must decide whether to flatten groups into one "Results" section while querying.

Card Rail

Craft/Notion-style discrete cards — whitespace repeated at every row, not just once.

The search field lives in its own header zone with a hairline beneath it, fully detached from the list (20px above and below). Each note is a rounded, elevated card using the existing "surface raised" tint, with real 10px gaps between cards — so the gap the user asked for shows up between every pair of notes, not only under the search field. Selection tints the whole card instead of the old solid-fill highlight.

Best forMaking "minimal" read as premium rather than empty. Trade-offDrops native `List`; needs custom keyboard row navigation.

Constellation ✦

The creative swing: a research timeline, not a file list.

A borderless, underline-only search field keeps the header almost invisible; a deliberate 32px gap opens onto a thin vertical spine running down the sidebar. Notes are dots along it — size and opacity fade with age — and the gap between two dots is proportional to the real time between those notes (log-compressed so old notes don't push the list off-screen). A large empty stretch of spine is itself information: "nothing happened here." It turns Noschen's own premise — thinking over time — into the sidebar's organizing idea, rather than a generic file browser.

Best forA distinctive, on-brand sidebar worth prototyping live before committing. Trade-offNeeds a strict accessibility fallback — see write-up — and gap-clamping so dense days don't collide.

Recommendation: ship Quiet Rail first — it directly answers the whitespace complaint with almost no risk and no data-model changes. Follow with Grouped Timeline once real note counts justify grouping. Keep Constellation as the deliberate creative bet: worth a throwaway prototype in the real app, since it's the one direction that turns Noschen's own premise — thinking over time — into the sidebar itself.
